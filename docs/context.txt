# Project Context

## Testing and coverage
- **Tests**: `npm run test` (via Docker: `docker compose run --rm dev npm run test`). Check test output for warnings (stderr) and fix or suppress expected ones.
- **Coverage**: `npm run test:coverage` writes report to `coverage/` (text summary, HTML, lcov). `coverage/` is gitignored. Run when touching tests or critical paths to keep track of coverage.

## Local / secrets (not committed, not in project)
- **GH_TOKEN**: Store outside the project so Cursor agents cannot read it. Example: create `~/.config/illo3d/gh.env` with one line `GH_TOKEN=ghp_...`. When running gh in the dev container, use: `docker compose run --rm --env-file "$HOME/.config/illo3d/gh.env" dev gh run list` (or `gh run view`). Do not put the token in a file inside the project; agents can read any file in the workspace.

## Open Conversations

### Translations (i18n)
- Structure: src/locales/en.json, es.json; I18nContext with t(key) and optional params. Only English filled for now; es.json has same keys (ready for Spanish later).
- User-facing strings via t(key); configurable values (currency, company name, etc.) from database/app state, not translation keys.

### Tech Stack Decisions
- **Framework**: React
- **Auth**: Google OAuth via Google Identity Services (GIS); "Sign in with Google" on LoginPage. Client ID from env (`VITE_GOOGLE_CLIENT_ID`). ID token stored for session; no backend verification. Access token for Drive API will be added when integrating Drive.
- **UI Library**: PrimeReact (installed and in use); Lara Light/Dark theme, user toggle, preference persisted. Atoms (Button, Input, Checkbox) and TabsLayout/DataTable use PrimeReact components.
- **App Type**: PWA (Progressive Web App)
- **Hosting**: GitHub Pages
- **Backend**: Google Drive API for storage
- **Kanban**: Need to select library (react-beautiful-dnd, dnd-kit, or react-trello)
- **Dev environment**: All dev tools only inside Docker; host needs only Docker (no npm/Node on laptop).
- **Design**: Colour palette in docs/design.md (logo-derived: green primary, blue secondary, orange accent; use with hierarchy).

### Backup strategy (data in Google Drive)
- **Scope**: database.json (+ optionally .3mf references or a manifest); no traditional server, so backups are user/Drive-centric.
- **We do the copies** — do not rely only on Drive version history; the app creates and manages backups.
- **Export from app** — yes. Button to download database.json (and optionally a list of linked .3mf paths) to the user's device.
- **Copy inside Drive** — yes. Timestamped copy (e.g. database.backup.2026-01-31.json) into a Backup folder in the same Drive. **Triggers (copy in Drive):** on demand (button), after save, others TBD (e.g. periodic, before major operations).
- **Import / restore** — yes. User can upload a previously exported or backup JSON to replace current data. No merge for now; merge may be considered later.

### App Overview
3D printing business management web app
- Hosted on GitHub Pages (static site)
- Google Drive as backend for configs and .3mf files
- No traditional backend

### Core Features Identified
1. **Layout and theme**
   - Side navigation to all pages; light and dark theme with user toggle (preference persisted)

2. **Inventory Management**
   - Track filaments: name, type, purchase date, grams, cost, price per gram, depleted status
   - Track other items: lights, keychains, accessories

3. **Sales History**
   - Record sales with date, description, price
   - Track materials used per sale (up to 4 colors with grams)
   - Calculate costs and profits

4. **Budget Calculator**
   - Calculate quotes without adding to sales history
   - Formula: VLOOKUP price per gram × grams × 1.1 markup
   - Support multiple colors per item

5. **Kanban Board**
   - Columns: pa pensare, PRECIO, POR HACER, HECHO, OBJETIVOS, DESCARATADO POR PRECIO
   - Track tasks/orders in progress

6. **Material Import**
   - Parse .3mf files for material usage (uploaded to Google Drive)
   - MakerWorld integration (v2)

7. **Analytics/Graphs**
   - v1: basic metrics (profits, sales over time)
   - v2: advanced (most profitable items, trends)
